# 2026-07-11

## Usage and quota design

Status: spec drafted, implementation not started.

Related work: PR #54 (`docs(usage): define quota accounting contract`).

### Goal

- 현재 tier quota와 live usage를 Settings에서 조회한다.
- 파일 쓰기마다 Space 전체를 `COUNT/SUM`하는 비용을 제거한다.
- Counter drift를 사용자 요청과 background reconciliation으로 복구한다.

### Generalized quota model

Quota는 `scope + metric + used + limit`으로 표현한다.

```text
Exact count       spaces, agents, API keys, connections, folder children
Stored counter    Space live nodes, Space live content bytes
Request limit     Text/File object size, text lines, metadata size
```

API는 계산 방식과 무관하게 `{used, limit}`을 반환한다. Persistence는 하나의 범용 metric table로 만들지 않고, 현재 고비용 scope에 맞는 `space_usage` typed table을 사용한다.

### Space usage implementation

```text
space_usage
  space_id PK/FK
  live_node_count
  live_content_bytes
  reconciled_at
  next_reconcile_at
```

- Space 생성 시 root node와 usage row를 같은 transaction에서 만든다.
- File-tree mutation은 `shared reconciliation gate -> Space -> space_usage` 순서로 잠근다.
- Reconciler는 Space별 exclusive gate를 try-lock하고 사용 중이면 건너뛴다.
- 원본 변경, usage delta, file change event는 같은 transaction에서 commit한다.
- Usage 조회와 quota 검사는 counter를 사용한다.
- 원본 node/text/file row는 reconciliation 기준이다.

### Reconciliation

- DB 전체에서 worker 하나만 활성화한다.
- 한 번에 Space 하나만 처리한다.
- `next_reconcile_at`을 약 7일 범위에 무작위 분산하고 due 순서로 처리한다.
- Busy Space는 건너뛰고 나중에 재시도한다.
- 재계산 중 해당 Space read는 허용하고 mutation만 일시적으로 거부한다.
- 사용자 요청은 `POST /api/v1/spaces/{space_id}/usage/reconcile`로 queue를 앞당긴다.
- 전체 재계산은 초기 backfill과 심각한 장애 복구용 관리자 작업으로만 둔다.

### Client behavior

- Refresh는 counter만 다시 조회한다.
- Recalculate는 owner가 특정 Space에만 요청한다.
- REST mutation 차단은 `503 usage_recalculation_in_progress`와 `Retry-After`를 반환한다.
- MCP read는 계속 동작하고 mutation은 같은 retryable kind를 반환한다.
- FE는 인증 상태와 편집 draft를 유지하며 mutation을 자동 재실행하지 않는다.

### Implementation order

1. Migration과 기존 Space backfill
2. Transactional counter shadow update와 mutation matrix test
3. Sequential reconciler와 manual queue endpoint
4. 기존 quota 검사 기준을 counter로 전환
5. `/api/v1/me/usage`와 Settings Usage UI
